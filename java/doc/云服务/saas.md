## 数据库设计

### 数据库设计的三范式

- 第一范式: 每一列都是不可分割的原子数据项
  - 每个字段里面的内容都是不可分割的原子数据项
- 第二范式: 在第一范式的基础上, 非主键列必须完全依赖于主键, 而不能依赖于主键的一部分
  - 一个表只做一个事情
- 第三范式: 在第二范式的基础上, 非主键列必须直接依赖于主键, 不能间接依赖于主键
  - 只要有一个 id 就够了
- 反三范式: 为了提高查询效率, 降低范式, 有时候会出现非主键列直接依赖于主键的情况
  - 适量而行
  - 空间换时间的操作
